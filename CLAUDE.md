# scLean Developer Guide

Memory-efficient single-cell RNA-seq analysis as a drop-in replacement for Seurat v5. Replaces Seurat's core computation with HDF5-backed disk computation supporting 500K+ cells on 8 GB RAM.

## Build System

Prerequisites: R >= 4.0, HDF5 >= 1.10.0, C++17 compiler, OpenMP (optional).

```bash
# First-time setup or after adding new .cpp files
./cleanup && ./configure

# Regenerate Rcpp bindings after C++ changes
Rscript -e 'Rcpp::compileAttributes()'

# Build and install
R CMD INSTALL .

# Run tests
Rscript -e 'devtools::test()'

# Build package tarball
R CMD build .

# Clean build artifacts
make clean
```

The `configure` / `configure.win` system detects HDF5 via Rhdf5lib's pkgconfig and checks for OpenMP support. C++17 is mandatory. `src/Makevars` is generated by configure and is **not tracked in git** — run `./cleanup && ./configure` after cloning or after adding new C++ source files.

**Key dependencies**: Seurat >= 5.0.0, SeuratObject >= 5.0.0, RcppEigen, RcppAnnoy (for bundled Annoy headers), Rhdf5lib. `leidenbase` is suggested but scLean bundles its own Leiden implementation in C++.

## Architecture

### Data Flow

```
R user code
  → S3 dispatch (.Seurat method → .scLeanAssay method)
    → RcppExports.R (.Call wrapper)
      → rcpp_io.cpp / rcpp_pipeline.cpp / rcpp_perf.cpp (R-to-C++ bridge)
        → Operator class (normalize/, scale/, pca/, etc.)
          → DataSource (logical data access, HDF5 paths centralized)
            → DiskMatrix → HDF5CSCMatrix / HDF5DenseMatrix
              → HDF5File (hdf5/)
          → ChunkScheduler (adaptive chunk sizing)
```

### S3 Dispatch Pattern

scLean intercepts Seurat's S3 generics without modifying Seurat's namespace:

1. `R/generics.R` re-exports Seurat generics (e.g., `NormalizeData`, `ScaleData`) via `@importFrom` + `@export`.
2. `R/zzz.R` captures the **original** Seurat S3 methods into the private environment `.seurat_original` during `.onLoad`.
3. `R/dispatch-helpers.R` provides shared helpers: `extract_sc_assay()` (extracts scLeanAssay from Seurat or returns directly), `resolve_chunk_size()` (fallback chain), `reinstall_assay()`, `register_layer()` (LogMap sync).
4. Each `.Seurat` method (e.g., `FindNeighbors.Seurat`) checks via `extract_sc_assay()` whether the default assay is a `scLeanAssay`. If yes, it calls `.scLeanAssay`. If no, it falls back to the stored original from `.seurat_original`.
5. Result: scLean objects get disk-backed computation; standard Seurat objects get standard Seurat computation. No namespace conflicts.

### HDF5 File Schema

See [HDF5-SCHEMA.md](HDF5-SCHEMA.md) for the full HDF5 file layout. Key conventions:

- Matrices stored as CSC (data/indices/indptr datasets)
- Feature/cell names as 1D string datasets
- Pipeline results under predictable paths: `/reductions/pca/`, `/graphs/nn/`, `/clusters/leiden/`
- ScaleData stores mean/SD only — PCA does centering on-the-fly via Lanczos

### Chunk Scheduling & Memory Model

The `ChunkScheduler` (in `src/utils/chunk_scheduler.cpp`) is the central memory management component:

- **Memory detection**: Cross-platform `detect_free_ram()` — on macOS, counts free + inactive pages.
- **Safety factor**: Only 50% of detected free RAM is assumed usable (accounts for HDF5 decompression buffers, Eigen temporaries, filesystem cache).
- **Operation multipliers**: Each operation type has a hardcoded multiplier (Normalize=3, Scale=6, VST=4, PCA=2.5, etc.) representing the peak temporary-to-read ratio.
- **3-Level OOM recovery**: When `std::bad_alloc` is caught, chunk size is halved up to 3 times (floor 128). The override persists for the operation's retry chain.
- **128 MB hard cap**: `MAX_DENSE_CHUNK_BYTES` prevents any single dense read from exceeding ~128 MB.
- **Auto mode**: When no chunk size override is set, the scheduler computes the optimal size from available RAM, operation type, and thread count.

### Thread Model

- Global thread count `g_num_threads` in `parallel.h` — plain global, set by `cpp_set_threads()`.
- Default is 1 for safety with non-threadsafe HDF5 builds.
- HDF5 read threads use thread-local file handles (`thread_local` map in `hdf5_file.cpp`).
- `ThreadGovernor` applies 5 rules at runtime to adjust thread count under resource pressure.
- Eigen threading is explicitly suppressed to avoid BLAS conflicts on macOS.

## Key Conventions

### R-to-C++ Integer Encoding

| R Value | C++ Integer | Module |
|---------|-------------|--------|
| `"LogNormalize"` | `0L` | Normalize |
| `"CLR"` | `1L` | Normalize |
| `"RC"` (RelativeCounts) | `2L` | Normalize |
| `"wilcox"` | `0L` | FindMarkers |
| `"t"` | `1L` | FindMarkers |
| `"LR"` (logistic regression) | `2L` | FindMarkers |
| `"leiden"` | `1L` | FindClusters |
| `"louvain"` | `0L` | FindClusters |
| Chunk auto-detect | `-1` | chunk-control |
| ident.2=NULL (all others) | `-1L` | FindMarkers |

### Sentinel Values

- `-1` → auto-detect chunk size in C++
- `-1L` → "test against all other clusters" in markers
- `NULL` → remove RAM cap (revert to auto-detect) in SetMaxRAM
- `0` for max_ram bytes → C++ auto-detects (no user cap set)
- `NA` → restore auto chunk size in SetChunkSize

### 0-Based vs 1-Based Indexing

C++ uses 0-based indexing. R uses 1-based indexing. The conversion (`- 1L`, `+ 1`) happens in R pipeline functions before calling C++ and after reading results back. Key locations: `pca.R:85`, `markers.R:95`, `io-convert.R` (passim).

### LogMap Registration

Seurat v5 Assay5 tracks which layers each cell/feature participates in via `LogMap` objects. When a pipeline step creates a new layer (e.g., "data" after NormalizeData), it must call `.add_layer_to_maps()` to synchronize the LogMaps. Failure to do so causes Seurat's internal validation to reject the object.

### Null-Coalescing Operator

`%||%` (defined in `R/utils.R`) chains fallback values: `a %||% b` returns `a` if not NULL, otherwise `b`. Used extensively for chunk size override chains: `user_provided %||% global_option %||% -1`.

## Unimplemented / Reserved Features

These parameters or functions exist in the API but are not yet implemented:

- `vars.to.regress` in ScaleData — covariate regression not implemented
- `in_memory` parameter in NormalizeData/ScaleData — in-memory path not implemented
- `write_cols` / `write_rows` in HDF5CSCMatrix — write-back not supported
- `loess_span` in VST — binned median approximation used instead of true LOESS
- Subset-restricted Lanczos PCA (>2000 genes) — always runs on full gene set
- Direct CellRanger .h5 streaming — loads entire matrix via Seurat::Read10X_h5() first
